# NexRemote Windows Host Privacy Policy

## 2. Product Scope

The Windows host application is intended to let you receive user-approved remote connections from compatible NexRemote clients for legitimate remote-control and device-management scenarios involving devices that you own, administer, or are expressly authorized to manage.

The Windows host does not require a Neural Nexus Studios account for its core operation. Neural Nexus Studios does not operate a cloud backend, analytics platform, advertising service, or crash-reporting backend for the normal use of this Windows host app.

## 3. Data Stored Locally On The PC

Depending on the features you enable and the way you use the app, the Windows host may store the following data locally on the PC:

- app settings, including PC name, ports, theme preference, remote-access preferences, audit-log preference, and legal acceptance timestamps
- trusted-device records, including device identifiers, device names, connection timestamps, and stored public keys
- local certificate files, certificate fingerprints, and related trust material used for secure transport
- local operational logs and, if enabled, local audit logs used for troubleshooting, diagnostics, and security review
- temporary local cache files used by app features such as QR presentation or companion tooling

## 4. Data Accessed Or Transmitted During Use

When you enable the Windows host and approve or use a connection, the app may access, display, or transmit the following data between your PC and a compatible NexRemote client:

- device names, device identifiers, connection metadata, network endpoints, and certificate fingerprints used for discovery, pairing, and trust
- keyboard, mouse, touchpad, touch, gamepad, joystick, trigger, d-pad, gyro, and other remote-input commands
- screen-sharing frames, display metadata, and optional system-audio stream data
- camera-streaming frames and camera metadata
- file and folder names, paths, metadata, and file contents used by the file-management feature
- clipboard contents when clipboard features are enabled and used
- task-manager and process information such as process names, IDs, and resource usage
- media metadata and playback state

Neural Nexus Studios does not receive this content as part of the app’s normal local operation. This data is intended to move only between the devices you choose to connect.

## 5. Capabilities And Sensitive Features

The Windows host declares or uses capabilities and behaviors that may access personal or sensitive information where relevant to its functionality, including:

- local network and internet communication for discovery, pairing, and remote sessions
- webcam access for camera enumeration and camera streaming
- full-trust desktop behavior required for packaged desktop-host functionality
- background availability, startup behavior, and local firewall configuration when enabled by the user
- access to local files and libraries for file-management and media-related features

If you do not grant the required consent, disable a feature, or if Windows privacy settings block access, the related feature should be unavailable or limited.

## 6. How The Windows Host Uses Data

The Windows host uses data only for the following purposes:

- discovering, authenticating, pairing, and maintaining connections to selected NexRemote clients
- delivering remote-control, streaming, file-management, clipboard, media, and task-management features requested by the user
- storing local settings, legal acknowledgements, trust records, and certificate material
- troubleshooting, diagnostics, and local security review

The Windows host is not designed to use your data for advertising, profiling, resale, cross-app tracking, or analytics monetization.

## 7. Sharing And Disclosure

Neural Nexus Studios does not sell personal or sensitive user data.

Neural Nexus Studios does not share NexRemote Windows host data with advertisers, analytics providers, or unrelated third parties for monetization.

Data may be disclosed only:

- to the NexRemote client device that you choose to connect as part of the app’s core function
- to platform or component providers only to the extent required for normal device or operating-system operation
- if required by applicable law, lawful process, or valid governmental request

## 8. Security

The Windows host uses local certificate material and related trust records to support secure connections where available. Certificate continuity and fingerprint validation help detect unexpected certificate changes.

However, security also depends on your configuration. If you intentionally use an insecure connection mode, expose the host on an untrusted network, or approve an untrusted device, your data may be more vulnerable to interception or misuse.

## 9. Retention

Under the current product behavior:

- settings, trusted-device records, certificate files, and legal acceptance timestamps remain stored locally until you change, remove, or delete them, or uninstall the app
- the operational log is compacted to approximately the most recent 30 days of entries
- if audit logging is enabled, audit-log files may remain stored locally until you delete them, disable logging and remove the file, or uninstall the app

## 10. Your Choices And Controls

You can:

- enable or disable local-network availability and background behavior
- review and remove trusted devices
- review Windows privacy settings, including webcam privacy settings
- disconnect active clients
- clear local app data, delete logs, and uninstall the app

Because the Windows host does not currently use developer-hosted user accounts for its core operation, there is generally no separate developer-side account database to delete.

## 11. Children

The Windows host is not directed to children under 13. Because the app can expose remote system controls, screens, files, clipboard content, audio, camera content, and process information, it is intended for teens and adults using devices they own or are authorized to manage.
